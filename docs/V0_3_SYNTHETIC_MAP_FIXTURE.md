# v0.3 Synthetic Map Fixture Layout

This document defines a tiny SoulForge-owned MSB-like map fixture layout for v0.3 parser plumbing work.

It is not a FromSoftware native MSB specification.

## Purpose

The fixture exists to prove that map export can carry:

- map IDs;
- entity IDs;
- entity names;
- entity kinds;
- positions and rotations;
- region IDs;
- region names;
- region shapes;
- region positions and sizes;
- high-confidence fixture metadata without claiming native authority.

## Common rules

- All integer fields are little-endian signed 32-bit values.
- Float fields are little-endian IEEE-754 float32 values.
- Strings are UTF-16LE null-terminated strings in the string pool.
- These fixtures may be generated by smoke scripts in temporary directories.
- They must never be treated as native game-format authority.

## Magic and marker

```text
0x00  4 bytes   ASCII magic: MSB\0
0x04  4 bytes   ASCII marker: SFMP
```

## Header

```text
0x08  int32     version, currently 1
0x0C  int32     entity count
0x10  int32     entity table start offset
0x14  int32     region count
0x18  int32     region table start offset
0x1C  int32     string pool start offset
```

## Entity row

Repeated `entity count` times. Row stride: 40 bytes.

```text
int32   entityId
int32   nameStringPoolRelativeUtf16Offset
int32   kindCode
float32 positionX
float32 positionY
float32 positionZ
float32 rotationX
float32 rotationY
float32 rotationZ
int32   reserved
```

`kindCode` values:

```text
0 = unknown
1 = character
2 = object
3 = asset
4 = collision
5 = mapPiece
```

## Region row

Repeated `region count` times. Row stride: 36 bytes.

```text
int32   entityId
int32   nameStringPoolRelativeUtf16Offset
int32   shapeCode
float32 positionX
float32 positionY
float32 positionZ
float32 sizeX
float32 sizeY
float32 sizeZ
```

`shapeCode` values:

```text
0 = unknown
1 = point
2 = sphere
3 = box
4 = cylinder
```

## Expected Bridge output

```text
resourceKind = map
parseStatus = partial
diagnostic code = MSB_SYNTHETIC_FIXTURE_CONFIRMED
```

Each entity should export:

```text
MapExport.entities[].entityId
MapExport.entities[].name
MapExport.entities[].kind
MapExport.entities[].position
MapExport.entities[].rotation
raw.confidence = high
raw.nativeFormatAuthority = false
```

Each region should export:

```text
MapExport.regions[].entityId
MapExport.regions[].name
MapExport.regions[].shape
MapExport.regions[].position
MapExport.regions[].size
raw.confidence = high
raw.nativeFormatAuthority = false
```

## Wire-up status

`SyntheticMapFixtureExports.cs` defines the parsing/export helper for this layout.

The intended route is:

```text
Program.cs export-map
  -> SemanticCandidateExports.TryExportMap
    -> SyntheticMapFixtureExports.TryExport first
    -> low-confidence visible-name fallback second
```

If direct edits to existing router files are blocked, Codex should perform this final wire-up locally and run smoke checks after build.
